# Public-Safety Boundary

This repository is intentionally curated for public portfolio use.

## Included

- Public-safe architecture notes.
- Public-safe screenshots from fake data.
- High-level migration decisions.
- Validation evidence summaries.
- Links to public GitHub, LinkedIn, Medium, and ng-openlayers assets.

## Excluded

- Private source code.
- Customer names.
- Company-specific internal terms.
- Production routes, endpoints, credentials, or infrastructure references.
- Real screenshots from internal systems.
- Private repository history.
- Internal DTO names or copied implementation details.
- Full reusable library implementation.

## Why The Boundary Exists

The public portfolio should show engineering judgment without exposing proprietary work. The useful signal for an employer is the architecture, migration strategy, validation discipline, and ability to communicate a complex modernization effort clearly.

The implementation can continue privately as a reusable PrimeNG-oriented library while this repository remains a clean public case study.
